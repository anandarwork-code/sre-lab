## Completed Work

### KVM Virtualization Setup
- Verified Intel VT-x support via /proc/cpuinfo and lscpu
- Installed KVM stack: qemu-kvm, libvirt, virt-install, libvirt-client
- Loaded kvm_intel kernel module
- Enabled and started libvirtd service
- Understood full stack: KVM → QEMU → libvirt → virsh

### VM Management
- Created VM1 (sre-vm1) using virt-install — text mode install
- Created VM2 (sre-db) using virt-install — text mode install
- Mastered virsh lifecycle: start, shutdown, destroy, reboot, autostart
- Created snapshot (snap1) on sre-vm1
- Configured SSH passwordless auth (ed25519) from host to both VMs

### User Management (sre-vm1)
- Created sradmin — full sudo via wheel group
- Created jradmin — limited sudo via /etc/sudoers (systemctl only)
- Created dev_user — no sudo access

### MySQL Database Setup (sre-db)
- Installed MySQL 8.0 on sre-db
- Ran mysql_secure_installation — root secured, anonymous users removed
- Created database: srelab
- Created user: sreapp@192.168.122.152 — locked to sre-vm1 only
- Granted full privileges on srelab.* to sreapp
- Opened firewall port 3306 for sre-vm1 access only

### Node.js App Stack (sre-vm1)
- Installed Node.js v20.20.1 and npm 10.8.2 via NodeSource repo
- Installed Nginx — enabled, running on port 80
- Built sreapp — Express HTTP server with / and /health endpoints
- /health endpoint verifies live MySQL connection on sre-db
- Created sreapp.service systemd unit — starts on boot, restarts on crash
- Configured Nginx as reverse proxy → forwards port 80 to Node.js port 3000
- Fixed SELinux blocking Nginx → Node.js: setsebool -P httpd_can_network_connect 1
- Full flow verified: curl port 80 → Nginx → Node.js:3000 → MySQL on sre-db

### Bash Scripting
- check_services.sh — loops through services, logs with timestamp, outputs X/3 count
- check_mysql.sh — checks mysqld via systemctl is-active, logs with tee -a
- check_sreapp.sh — full stack health check: MySQL via HTTP, nginx, sreapp via systemctl
